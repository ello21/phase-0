Version Control for Bosses

Picture this: You're on a team of 3 boss-level developers in the midst of a caffeine-inspired-all-nighter to get your STATUS-QUO-DISRUPTING app deployed to the Google Play Store by the next morning. Does the app work? Yes. Are all who look upon it driven to tears? Hardly.

Your goal for the night was just to add that final design touch to take your app from basic to "BALLINNNNNN!"...and your dreams of breaking the internet were well within your reach...until...James broke the code.

No problem, git to the rescue!!

Oh...you DIDN'T use version control because it "slowed you down"? You thought git was for "noobs"?

Well, the rest of this post is for those of us who won't be picking up the broken pieces of our lives for the next week.

Version control is what gives developers the flexibility to experiment and break stuff without fear. Any good version control system will save and track changes to a project over time, so that when a faulty piece of code is introduced, the team can revert back to a previous working version of the project and continue on their road to stardom.

Git is a version control system with some particularly useful features. One is that each team member can have a seperate but complete version of the project on their local computer, in addition to a central repository that the team may host on a remote server or cloud, such as GitHub. Having this distributed workflow provides a safety net for times when local computers crash. The entire project's codebase will be safely hosted in the cloud and can be accessed from any computer.

Git workflow includes steps to create a local repo, add files it, and then begin tracking and "committing" changes made to those files. At any time, a user can revert back to previous file versions, or, once changes are final, the user can "push" those changes into the team's master repo. An added feature that GitHub provides is the "Pull Request" which makes it easy for other team members to review code before it's "pulled" into the master project files.

All these features make Git and GitHub not just useful, but necessary, for any development team. Noobs and bosses alike.
